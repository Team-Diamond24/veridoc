Core processing layers

OCR & ingestion

Layer 01

PaddleOCR-VL-0.9B converts scanned pages into structured Markdown, preserving table cell coordinates. Every extracted value carries a page and bounding-box reference so the frontend can highlight the exact PDF line a verdict cites.

model PaddleOCR-VL-0.9B
languages 109 incl. Hindi Devanagari
benchmark SOTA · OmniDocBench v1.5
cost $0.09 / 1K pages self-hosted
output Markdown + bounding boxes

Obligation extraction

Layer 02

LegalBERT fine-tuned on government RFP corpora parses each criterion into typed obligation nodes. Dependencies between criteria are modelled as directed edges — a failing upstream node cascades failure downward automatically.

model LegalBERT (EMNLP 2020)
node fields modal · threshold · temporal · evidence · attestation
graph type Directed acyclic
modal types MANDATORY · CONDITIONAL · OPTIONAL

Multi-hop evidence mapper

Layer 03

IndexRAG builds bridging facts at index time — linking a turnover claim in the bid letter to the audited P&L, the audit period statement, and the CA certificate. A single-pass retrieval returns the full cross-document chain.

method IndexRAG (arXiv 2603.16415)
gain +4.6 F1 over naive RAG
chain depth 2–4 exhibits per criterion
output Evidence chain + completeness score

Verdict engine

Layer 04

Scores each criterion on three independent axes, combined as a weighted product — not a sum. A zero on any axis vetoes the result regardless of the other two. Claude Sonnet 4 reasons over the assembled evidence chain to produce JSON-structured verdicts.

axis 1 Evidence quality — legible, complete, formatted
axis 2 Semantic match — addresses the right obligation
axis 3 Threshold compliance — value clears the bar
combination Weighted product (not sum)

Parallel modules

Counterfactual explainer

audit trail

For every failing criterion, generates a precise recourse statement. Not "you failed" — "if you submit X covering period Y attested by Z, this criterion passes." Required under GFR 2017 Rule 204.

basis Coston et al. FAT* 2020 · Leben CMU 2023
output Per-criterion recourse statement
legal ref GFR 2017 Rule 204

Collusion risk detection

advisory only

CNN scans bidder interaction graphs across the full bid pool. Six anomaly detectors run in parallel. Flags are advisory — no bid is blocked automatically. The officer decides.

model CNN · García Rodríguez et al.
accuracy 81–95% (OECD documented)
checks Price · timing · address · text · pattern

Active learning loop

improves over time

Every evaluator override is captured as a structured correction: which axis was wrong, correct label, PDF span, obligation type. Corrections accumulate into a fine-tuning dataset. Threshold: 200 corrections triggers a scheduled retrain.

reference Singapore iCAS: 61% → 88% in 18 months
format Structured form, not free text
threshold 200 new corrections → retrain queue
Active learning cycle
01
VERIDOC produces verdict
Criterion scored on 3 axes. Borderline cases flagged for review automatically.
02
Officer reviews and overrides
Structured correction form captures which axis was wrong and the correct classification.
03
Correction stored with context
Original prediction, corrected label, PDF bounding box, obligation type — all saved together.
04
Periodic fine-tune
At 200-correction threshold, obligation classifier and verdict engine are fine-tuned on accumulated corrections.
